📚 Complete Terminology Guide

IDPS (Intrusion Detection and Prevention System)

A comprehensive security tool that monitors network traffic for suspicious activity, detects potential threats, and can take automated action to prevent intrusions. IDPS systems combine detection capabilities with the ability to block or prevent detected attacks in real-time.

Core Concept Lecture 1

IDS (Intrusion Detection System)

A monitoring tool that detects and generates alerts for suspicious or unauthorized network activity. Unlike an IPS, an IDS does not actively block traffic but serves as a detection and alerting mechanism, providing visibility into security threats.

Detection Only Lecture 1

IPS (Intrusion Prevention System)

A security system that is typically placed inline with network traffic to actively detect, alert, and block potential threats in real-time. Unlike an IDS, an IPS takes proactive action to prevent attacks from succeeding.

Active Prevention Lecture 1

NIDPS (Network-Based Intrusion Detection and Prevention System)

An IDPS that monitors network traffic at strategic points in the network, such as routers, firewalls, and network segments. NIDPS analyzes packets and flows but cannot inspect encrypted traffic like HTTPS due to network-level limitations.

Network Level Deployment

HIDPS (Host-Based Intrusion Detection and Prevention System)

An IDPS deployed on individual hosts or servers. It monitors system logs, file access, registry changes, and local processes. HIDPS can detect attacks targeting the host and inspect encrypted traffic from that host's perspective.

Host Level Deployment

WIDPS (Wireless Intrusion Detection and Prevention System)

A specialized IDPS designed for wireless environments. WIDPS uses dedicated sensors to capture and analyze wireless traffic, specifically detecting threats like Rogue Access Points (Rogue APs), Evil Twin attacks, and other wireless-specific threats.

Wireless Specialized

Hybrid IDPS

An IDPS architecture that combines multiple IDPS types (Network-Based, Host-Based, and Wireless) to provide comprehensive coverage across the entire IT environment. Hybrid systems offer the most complete protection.

Combined Advanced

Signature-Based Detection

A detection methodology that relies on predefined patterns or "signatures" of known threats stored in a signature database. When traffic matches a known signature, an alert is generated. This approach is effective against known attacks but cannot detect zero-day exploits.

Known Threats Database-Driven

Anomaly-Based Detection

A detection methodology that establishes a baseline of normal network or system behavior and flags significant deviations from that baseline as potential threats. This approach can detect novel attacks but may generate false positives.

Behavioral Adaptive

Stateful Protocol Analysis

A detection methodology that analyzes the state of network protocols and their expected behavior. It compares observed protocol behavior against known, valid protocol states and flags deviations as potential attacks.

Protocol-Focused Advanced

Heuristic-Based Detection

A detection methodology that combines stateful protocol analysis, signature-based detection, and anomaly-based detection to provide comprehensive threat detection. It uses multiple analysis engines working together.

Combined Approach Comprehensive

Baseline (Anomaly Detection)

The established profile of normal network or system behavior used in anomaly-based detection. The system regularly updates and refines the baseline to maintain accuracy and reduce false positives.

Reference Point Anomaly-Based

Threshold (Anomaly Detection)

The predefined deviation level that triggers an alert in anomaly-based detection systems. When behavior deviates beyond the threshold from the baseline, it's flagged as suspicious.

Alert Trigger Configuration

Bypassing

An attack technique where adversaries attempt to circumvent security mechanisms without distorting network traffic. This involves exploiting system vulnerabilities or gaps in monitoring to avoid detection entirely.

Evasion Type Advanced Attack

Evasion

A set of techniques used by attackers to hide their attacks or alter their appearance so they are not detected by security systems such as firewalls or IDPS. Evasion techniques modify how attacks appear to security systems.

Obfuscation Attack Technique

Packet Fragmentation (Evasion Technique)

An evasion technique where an attacker splits a malicious payload into many small packets to avoid detection by an IDPS. The IDPS may fail to detect the attack until the packets are reassembled.

Fragmentation Evasion

Signature Mutation

An evasion technique where polymorphic malware modifies its attack code so that it doesn't match known patterns in a signature database. This allows attacks to evade signature-based detection systems.

Polymorphic Malware Evasion

IP Spoofing & MAC Spoofing

Evasion techniques where attackers forge or spoof the source IP address or MAC address to mask their identity and location, making it difficult to trace attacks back to their origin.

Spoofing Identity Masking

Blind Spot Exploitation

A bypassing technique where attacks are launched through unmonitored channels such as USB, Wi-Fi Direct, or other network segments not covered by IDPS sensors, allowing attacks to bypass detection entirely.

Physical Bypass Bypassing

Fileless Attacks

Bypass techniques where malware operates entirely in memory without writing to disk, making it difficult for file-based IDPS detection mechanisms to identify the threat.

In-Memory Advanced Bypass

Encryption & Tunnelling

Evasion techniques where attackers hide their traffic inside encrypted connections or tunnels, making it impossible for network-based IDPS to inspect the payload without decryption capabilities.

Encryption Traffic Hiding

Timing Attacks

Evasion technique where attackers send attacks during specific times (late-night hours, low-traffic periods) when monitoring may be less active or when detection thresholds are less sensitive.

Timing-Based Evasion

Protocol Misuse

Evasion technique where attackers exploit protocol specifications in unexpected ways, leveraging valid protocol behavior for malicious purposes in ways that bypass detection systems.

Protocol-Based Advanced Evasion

SIEM (Security Information and Event Management)

A centralized system that collects, analyzes, and correlates security events from multiple sources—including IDPS, firewalls, and servers—to provide a unified security view and enable better threat detection and response.

Centralized Log Management

CIA Triad

A fundamental security model consisting of Confidentiality, Integrity, and Availability. IDPS systems help protect all three: Confidentiality (preventing unauthorized access), Integrity (preventing data modification), and Availability (preventing denial-of-service attacks).

Foundational Security Model

False Positive

A false positive occurs when an IDPS generates an alert for activity that is not actually malicious. Minimizing false positives requires proper rule tuning and sensitivity adjustments to prevent alert fatigue.

Alert Accuracy Configuration

Deep Packet Inspection (DPI)

An advanced packet filtering and analysis method that examines the payload data of packets, not just headers. Often used with SSL/TLS decryption to inspect encrypted traffic for threats.

Advanced Analysis Inspection

Security Incident

Any unauthorized activity or event that negatively affects the confidentiality, integrity, or availability of information or systems. Security incidents range from malware infections to data breaches.

Event Classification Response

Multi-Layer Inspection

A defense strategy that combines multiple IDPS types and detection methods (Network-Based, Host-Based, Anomaly-Based, Signature-Based) to provide comprehensive coverage and defense in depth.

Defense Strategy Advanced

Packet Reassembly

An anti-evasion technique that involves collecting and reassembling fragmented packets before analysis. This prevents attackers from using packet fragmentation to evade detection.

Anti-Evasion Fragment Handling

SSL/TLS Inspection

An anti-evasion strategy that decrypts encrypted traffic for Deep Packet Inspection. This allows IDPS to see inside encrypted connections that would otherwise be opaque to network monitoring.

Encryption Handling Advanced Defense

Snort

A popular open-source IDS/IPS that is lightweight and flexible. Snort is known for being multi-threaded and supporting TLS/SSL decryption, though it lacks a built-in graphical user interface. It's highly customizable through rules.

Open-Source Lightweight

Suricata

An open-source IDPS noted for multi-threading support and faster traffic processing compared to Snort. Suricata was designed as a high-performance successor with modern capabilities.

Open-Source High-Performance

Zeek (formerly Bro)

A network monitoring platform known for detailed logging and behavioral analysis rather than traditional signature-based detection. Zeek provides comprehensive network intelligence.

Open-Source Behavioral

Cisco Firepower

A commercial IDPS solution that utilizes the Talos Threat Intelligence platform for up-to-date threat data. Firepower provides advanced threat protection with real-time threat intelligence.

Commercial Enterprise

IBM QRadar

A commercial SIEM platform that collects and analyzes security events from multiple sources. QRadar provides threat intelligence and incident response capabilities.

SIEM Commercial

Splunk

A commercial platform for searching, monitoring, and analyzing machine data including security events. Splunk can aggregate and correlate IDPS logs for better security visibility.

SIEM Log Analysis

🏗️ IDPS Component Architecture

  1. Sensors/Agents: Deployed on networks or hosts to collect data from system logs, network traffic, or host activities in real-time
  2. Analysis Engine: Processes collected data and applies detection methodologies (signature, anomaly, protocol, or heuristic-based)
  3. Management Console: Provides centralized control, configuration, and monitoring interface for administrators
  4. Alert System: Generates and delivers alerts for detected threats through various channels (email, syslog, SIEM)
  5. Database: Stores signatures, baselines, protocol definitions, or other detection parameters
  6. Response Module: Optional component that automatically takes action (blocking, isolation) on detected threats

📋 Comprehensive Lists & Topics

🔷 IDPS Types & Deployment Models

  1. NIDPS (Network-Based IDPS): Deployed at network perimeter, routers, and firewalls. Monitors all network traffic but cannot inspect encrypted data like HTTPS. Best for detecting external attacks.
  2. HIDPS (Host-Based IDPS): Installed on individual systems and servers. Monitors system logs, file access, and local processes. Can inspect encrypted traffic and detect insider threats.
  3. WIDPS (Wireless IDPS): Monitors wireless networks using dedicated sensors. Detects Rogue APs, Evil Twin attacks, and other wireless-specific threats. Essential for securing WiFi infrastructure.
  4. Hybrid IDPS: Combines multiple IDPS types for comprehensive coverage. Provides the most complete protection across network, host, and wireless environments.

📍 Deployment Modes

  • Promiscuous Mode: Passive monitoring without affecting traffic (typical for IDS)
  • Inline Mode: Active filtering and blocking of traffic (typical for IPS)
  • Passive Mode: Monitoring without transmission of data

🔍 Detection Methodologies

  1. Signature-Based Detection: Matches traffic against known attack signatures. Fast and accurate for known threats but cannot detect zero-day exploits.
  2. Anomaly-Based Detection: Identifies deviations from normal behavior. Can detect novel attacks but requires proper baseline tuning to minimize false positives.
  3. Stateful Protocol Analysis: Validates protocol behavior against expected states. Detects protocol exploits and deviations from proper protocol usage.
  4. Heuristic-Based Detection: Combines multiple detection methods (signature, anomaly, and protocol analysis) for comprehensive protection.

📝 IDPS Log Contents & Fields

  • Timestamp: Exact date and time of the event
  • Protocol: Type of protocol used (TCP, UDP, ICMP, HTTP, etc.)
  • Source IP Address: IP address of the originating host
  • Source Port: Port number from the originating host
  • Destination IP Address: IP address of the target host
  • Destination Port: Port number on the target host
  • Signature ID: Identifier for the matched attack signature
  • Alert Message: Description of the detected threat
  • Priority Level: Severity rating (1=High, 2=Medium, 3=Low)
  • Payload: Optional packet data that triggered the alert

🎯 Log Examples

Format: [Timestamp] [Source IP:Port] → [Dest IP:Port] [Protocol] [Signature ID] [Message] [Priority]

Example 1: [2024-01-15 14:23:45] 192.168.1.100:52341 → 10.0.0.50:22 TCP SID:001234 "SSH Brute Force Attack Detected" Priority:1

Example 2: [2024-01-15 14:24:12] 203.0.113.50:80 → 192.168.1.50:12345 TCP SID:005678 "SQL Injection Attempt" Priority:1

🚀 Bypassing Techniques

  1. Blind Spot Exploitation: Attacking through unmonitored channels (USB ports, Wi-Fi Direct, infrared, physical connections)
  2. Fileless Attacks: Malware executes entirely in memory without touching disk, avoiding file-based detection
  3. Rule Evasion: Crafting attacks to bypass specific IDPS rules and policies
  4. Physical Bypass: Physically circumventing network security controls (removing cables, installing backdoors)

⚡ Evasion Techniques

  1. Packet Fragmentation: Splitting packets to sizes IDPS cannot process, forcing reassembly
  2. Encryption & Tunnelling: Hiding attack traffic inside encrypted connections (VPN, TLS)
  3. IP Spoofing & MAC Spoofing: Forging source addresses to hide attacker identity
  4. Signature Mutation: Modifying attack code to change its signature/pattern
  5. Timing Attacks: Sending attacks during specific times (late night, weekends, low-traffic periods)
  6. Protocol Misuse: Exploiting protocol specifications in unexpected ways
  7. Polymorphic Techniques: Self-modifying code that changes each execution

🛡️ Anti-Evasion Strategies

  1. Regular Rule & Signature Updates: Keep signature databases current with latest threat patterns and CVE information
  2. Behavioral Analysis: Implement anomaly-based detection to identify deviations from normal behavior
  3. Packet Reassembly: Reassemble fragmented packets before analysis to detect hidden attacks
  4. SSL/TLS Inspection: Decrypt and inspect encrypted traffic using Deep Packet Inspection
  5. Multi-Layer Inspection: Combine Network-Based IDS with Host-Based IDS for comprehensive coverage
  6. Security Awareness Training: Educate users to avoid suspicious links and unknown files
  7. Protocol Validation: Enforce strict protocol compliance and reject deviations
  8. Threat Intelligence Feeds: Integrate up-to-date threat intelligence (Talos, AlienVault, etc.)

🔧 Common SIEM Tools

  1. IBM QRadar: Enterprise SIEM with advanced threat intelligence and correlation
  2. Splunk: Powerful log analysis and search platform for security data
  3. ArcSight: HPE ArcSight provides real-time security monitoring and incident response
  4. ELK Stack: Elasticsearch, Logstash, Kibana - open-source SIEM alternative
  5. Microsoft Sentinel: Cloud-native SIEM and SOAR platform

🔓 Open-Source IDPS/IDS Tools

  1. Snort: Lightweight, multi-threaded, supports TLS/SSL decryption. No built-in GUI
  2. Suricata: Multi-threaded, faster processing than Snort, TLS/SSL support
  3. Zeek (formerly Bro): Network monitoring platform with detailed logging and behavioral analysis
  4. OSSEC: Host-based intrusion detection with file integrity checking
  5. Tripwire: File integrity monitoring and change detection

💼 Commercial IDPS Solutions

  • Cisco Firepower: Uses Talos Threat Intelligence platform for real-time updates
  • Palo Alto Networks Threat Prevention: Advanced threat protection with machine learning
  • Fortinet FortiGate: Unified security appliance with IDPS capabilities
  • Check Point IDPS: Enterprise-grade intrusion detection and prevention

🚨 SANS Incident Response Lifecycle

  1. Preparation: Set up tools, training, and processes for incident response. Include IDPS, SIEM, and communication channels
  2. Detection & Analysis: Identify and analyze security incidents. Use IDPS alerts and logs to confirm incidents
  3. Containment: Isolate affected systems to prevent the attack from spreading. Can be short-term (temporary blocking) or long-term (sustained isolation)
  4. Eradication: Remove the threat from the environment. Remove malware, close vulnerabilities, change compromised credentials
  5. Recovery: Restore systems to normal operations. Patch systems, restore from clean backups, reactivate systems
  6. Lessons Learned: Review and improve processes. Conduct post-mortem analysis and update security controls

📊 Security Incident Classification

  • Malware Infection: Detection and removal of malicious software
  • Unauthorized Access: Compromised accounts or unauthorized system access
  • Data Breach: Unauthorized access or exfiltration of sensitive data
  • DoS/DDoS Attack: Denial of service attacks impacting availability
  • Insider Threat: Malicious or negligent insider activity
  • Configuration Mistake: Unintentional security misconfigurations
