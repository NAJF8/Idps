📚 Key Definitions

IDPS (Intrusion Detection and Prevention System)

A security tool that monitors network traffic for suspicious activity, detects potential threats, and can take automated action to prevent intrusions. IDPS systems combine detection capabilities with the ability to block or prevent detected attacks.

IDS (Intrusion Detection System)

A monitoring tool that detects and generates alerts for suspicious or unauthorized network activity. Unlike an IPS, an IDS does not actively block traffic but serves as a detection and alerting mechanism.

IPS (Intrusion Prevention System)

A security system that is typically placed inline with network traffic to actively detect, alert, and block potential threats in real-time. Unlike an IDS, an IPS takes proactive action to prevent attacks.

NIDPS (Network-Based Intrusion Detection and Prevention System)

An IDPS that monitors network traffic at strategic points in the network, such as routers, firewalls, and network segments. NIDPS analyzes packets and flows but cannot inspect encrypted traffic like HTTPS.

HIDPS (Host-Based Intrusion Detection and Prevention System)

An IDPS deployed on individual hosts or servers. It monitors system logs, file access, and local processes. HIDPS can detect attacks targeting the host and inspect encrypted traffic from that host's perspective.

WIDPS (Wireless Intrusion Detection and Prevention System)

A specialized IDPS designed for wireless environments. WIDPS uses dedicated sensors to capture and analyze wireless traffic, specifically detecting threats like Rogue Access Points (Rogue APs) and Evil Twin attacks.

Signature-Based Detection

A detection methodology that relies on predefined patterns or "signatures" of known threats stored in a signature database. When traffic matches a known signature, an alert is generated. This approach is effective against known attacks but cannot detect zero-day exploits.

Anomaly-Based Detection

A detection methodology that establishes a baseline of normal network or system behavior and flags significant deviations from that baseline as potential threats. This approach can detect novel attacks but may generate false positives.

Stateful Protocol Analysis

A detection methodology that analyzes the state of network protocols and their expected behavior. It compares observed protocol behavior against known, valid protocol states and flags deviations as potential attacks.

Heuristic-Based Detection

A detection methodology that combines stateful protocol analysis, signature-based detection, and anomaly-based detection to provide comprehensive threat detection. It uses multiple analysis engines working together.

Bypassing

An attack technique where adversaries attempt to circumvent security mechanisms without distorting network traffic. This involves exploiting system vulnerabilities or gaps in monitoring to avoid detection.

Evasion

A set of techniques used by attackers to hide their attacks or alter their appearance so they are not detected by security systems such as firewalls or IDPS. Evasion techniques modify how attacks appear to security systems.

SIEM (Security Information and Event Management)

A centralized system that collects, analyzes, and correlates security events from multiple sources—including IDPS, firewalls, and servers—to provide a unified security view and enable better threat detection and response.

Security Incident

Any unauthorized activity or event that negatively affects the confidentiality, integrity, or availability of information or systems. Security incidents range from malware infections to data breaches.

Packet Fragmentation (Evasion Technique)

An evasion technique where an attacker splits a malicious payload into many small packets to avoid detection by an IDPS. The IDPS may fail to detect the attack until the packets are reassembled.

Signature Mutation

An evasion technique where polymorphic malware modifies its attack code so that it doesn't match known patterns in a signature database. This allows attacks to evade signature-based detection systems.

IP Spoofing & MAC Spoofing

Evasion techniques where attackers forge or spoof the source IP address or MAC address to mask their identity and location, making it difficult to trace attacks back to their origin.

Blind Spot Exploitation

A bypassing technique where attacks are launched through unmonitored channels such as USB, Wi-Fi Direct, or other network segments not covered by IDPS sensors, allowing attacks to bypass detection.

Fileless Attacks

Bypass techniques where malware operates entirely in memory without writing to disk, making it difficult for file-based IDPS detection mechanisms to identify the threat.

CIA Triad

A fundamental security model consisting of Confidentiality, Integrity, and Availability. IDPS systems help protect all three elements: Confidentiality (preventing unauthorized access), Integrity (preventing data modification), and Availability (preventing denial-of-service attacks).

False Positive

A false positive occurs when an IDPS generates an alert for activity that is not actually malicious. Minimizing false positives requires proper rule tuning and sensitivity adjustments.

Snort

A popular open-source IDS/IPS that is lightweight and flexible. Snort is known for being multi-threaded and supporting TLS/SSL decryption, though it lacks a built-in graphical user interface.

IDPS Component Architecture

  1. Sensors/Agents: Collect data from system logs, network traffic, or host activities
  2. Analysis Engine: Processes collected data and applies detection methodologies
  3. Management Console: Provides centralized control and configuration interface
  4. Alert System: Generates and delivers alerts for detected threats
  5. Database: Stores signatures, baselines, or protocol definitions

📋 Key Lists & Enumerations

IDPS Types & Deployment Models

  1. NIDPS (Network-Based IDPS): Deployed at network perimeter, routers, and firewalls
  2. HIDPS (Host-Based IDPS): Installed on individual systems and servers
  3. WIDPS (Wireless IDPS): Monitors wireless networks with dedicated sensors
  4. Hybrid IDPS: Combines multiple IDPS types for comprehensive coverage

IDPS Log Contents

  • Timestamp of the event
  • Protocol used (TCP, UDP, ICMP, etc.)
  • Source IP address and Port
  • Destination IP address and Port
  • Signature ID
  • Alert Message
  • Priority Level

Detection Methodologies

  1. Signature-Based Detection: Matches traffic against known attack signatures
  2. Anomaly-Based Detection: Identifies deviations from normal behavior
  3. Stateful Protocol Analysis: Validates protocol behavior against expected states
  4. Heuristic-Based Detection: Combines multiple detection methods

Bypassing Techniques

  1. Blind Spot Exploitation: Attacking through unmonitored channels (USB, Wi-Fi Direct)
  2. Fileless Attacks: Malware that executes entirely in memory
  3. Rule Evasion: Bypassing specific IDPS rules and policies
  4. Physical Bypass: Physically bypassing network security controls

Evasion Techniques

  1. Packet Fragmentation: Splitting packets to avoid signature detection
  2. Encryption & Tunnelling: Hiding traffic inside encrypted connections
  3. IP Spoofing & MAC Spoofing: Forging source addresses
  4. Signature Mutation: Modifying code to avoid signature matching
  5. Timing Attacks: Sending attacks during low-traffic periods or specific times
  6. Protocol Misuse: Exploiting protocol specifications unexpectedly

Anti-Evasion Strategies

  1. Regular Rule & Signature Updates: Keep signature databases current
  2. Behavioral Analysis: Implement anomaly-based detection
  3. Packet Reassembly: Reassemble fragmented packets before analysis
  4. SSL/TLS Inspection: Decrypt and inspect encrypted traffic (DPI)
  5. Multi-Layer Inspection: Combine NIDPS and HIDPS
  6. Security Awareness Training: Educate users about threats

Common SIEM Tools

  1. IBM QRadar
  2. Splunk
  3. ArcSight
  4. ELK Stack
  5. Microsoft Sentinel

SANS Incident Response Lifecycle

  1. Preparation: Set up tools and processes for incident response
  2. Detection & Analysis: Identify and analyze security incidents
  3. Containment: Isolate affected systems to prevent spread
  4. Eradication: Remove the threat from the environment
  5. Recovery: Restore systems to normal operations
  6. Lessons Learned: Review and improve processes

Open-Source IDPS/IDS Tools

  1. Snort: Lightweight, multi-threaded, supports TLS/SSL decryption
  2. Suricata: Multi-threaded, faster processing, TLS/SSL support
  3. Zeek (formerly Bro): Network monitoring with detailed logs and behavioral analysis
  4. OSSEC: Host-based intrusion detection
  5. Tripwire: File integrity checking

Commercial IDPS Solutions

  • Cisco Firepower: Uses Talos Threat Intelligence platform
  • Palo Alto Networks Threat Prevention
  • Fortinet FortiGate
  • Check Point IDPS

IDPS Deployment Modes

  • Promiscuous Mode: Passive monitoring without affecting traffic (IDS)
  • Inline Mode: Active filtering of traffic (IPS)
  • Passive Mode: Monitoring without transmission
